Привет! Мы начинаем реализацию многоуровневой (MLM) партнерской программы для нашего проекта. Нам нужно спроектировать и написать логику базы данных в Supabase (PostgreSQL) и связать её с фронтендом и Telegram-ботом.

Выступи в роли Senior Backend & Web3 Developer. Вот утвержденная архитектура нашего проекта:

### 1. Архитектурный стек:
* **Фронтенд:** React (Vite) + Tailwind CSS + Web3-библиотеки (wagmi / viem / RainbowKit) для подключения кошельков.
* **Бэкенд & БД:** Supabase (PostgreSQL) + Edge Functions (TypeScript).
* **Интерфейсы:** Веб-дашборд + Telegram-бот (Node.js/Python), работающий как интерфейс к Supabase.
* **Блокчейн:** Сеть Base (EVM L2). 
* **Модель расчетов:** Гибридная ("off-chain accounting, on-chain settlement"). В блокчейне живет только стандартный, максимально простой токен ERC-20 от OpenZeppelin без кастомной логики. Вся MLM-структура, балансы, уровни и начисления считаются на стороне Supabase в реальном времени.

### 2. Бизнес-логика партнерской программы:
* При регистрации пользователя по реферальной ссылке, в его профиле жестко фиксируется ID его пригласителя (спонсора/аплайна).
* Пользователи могут покупать партнерские уровни/статусы за токены. 
* Оплата уровня происходит путем списания с их внутреннего баланса в Supabase (который пополняется через депонирование токенов на наш центральный Treasury-адрес на Base).
* В момент покупки уровня бэкенд должен автоматически рассчитать и распределить проценты по цепочке пригласителей наверх (например: 1-й уровень — X%, 2-й уровень — Y% и т.д.) и обновить их внутренние балансы.
* Вывод средств (Withdraw) происходит по запросу пользователя: бэкенд проверяет его внутренний баланс в Supabase и инициирует одну прямую транзакцию отправки токенов с нашего Treasury-кошелька на адрес пользователя.

### 3. Твоя первая задача:
Спроектируй структуру таблиц (схему БД) в Supabase для реализации этой логики. Нам нужны:
1. Таблица профилей (`profiles`) со связью реферал-пригласитель.
2. Таблица внутренних балансов и логов транзакций (`balances` / `transactions`).
3. Таблица истории партнерских начислений (`referral_rewards`).

Напиши SQL-код (или миграции) для создания этих таблиц с учетом внешних ключей (Foreign Keys) и индексов для быстрого поиска по реферальным веткам.
